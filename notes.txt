Anatomía de un JWT

Un JWT se compone de 3 partes principales, generalmente divididas por un punto:
  -Header.
  -Payload.
  -Signature.

Header:
  -typ: El tipo, que en este caso siempre será "JWT".
  -alg: Algoritmo usado para encriptar el signature. Puede ser asincrono o sincrono.
    .Asincronos: Usan 2 llaves de encriptación, la publica y la privada, donde la pública se usa para encriptar,
      y la privada para desencriptar.
    .Sinronos: Se usa la misma llave para encriptar y desencriptar.
    .Ambos son seguros de usar, pero cambian en donde los vamos a usar:
      -Asincronos: Deben usarse donde haya partes públicas que puedan tener acceso a esta llave.
      -Sincronos: Solo deben usarse en sistemas como el backend.

Payload:
  -Donde guardamos toda la info. de nuestro usuario, incluso scopes de autorización.
  -Este payload se compone de algo llamado 'claims', representados por 3 letras para mantener el JWT pequeño.
  -Hay diferentes tipos de claims:
    .Registered claims: Ya tienen una definición propia y deben respetarse.
    .Public claims: Una lista de claims que pueden usar diferentes apps y ya están definidos.
    .Private claims: Claims que nosotros definimos para nuestra app.

Signature o firma:
  -Se compone del header codificado + el payload codificado.
  -A todo esto se le aplica el alg. de encriptación usando un 'Secret'.
